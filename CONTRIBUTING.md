Guia de Contribuição (CONTRIBUTING)

Agradecemos o seu interesse em contribuir para o Clone do Moodle! Para manter a qualidade do código e a consistência do projeto, siga as diretrizes abaixo.

1. Processo de Desenvolvimento

Faça um Fork do repositório.

Crie uma Branch para a sua funcionalidade (feature/nome-da-feature) ou correção (fix/nome-do-bug).

Desenvolva o código seguindo as normas de estilo (PEP 8 para Python).

Certifique-se de que todos os testes passam.

Submeta um Pull Request (PR) detalhando as alterações.

2. Normas de Código

Comentários: Devem ser escritos em Português.

Commits: Utilize mensagens claras e no imperativo (ex: "Adiciona autenticação JWT" em vez de "adicionando as coisas").

Documentação: Qualquer nova funcionalidade deve ser refletida nos ficheiros .md correspondentes.

3. Estrutura de Branches

main: Código estável em produção.

develop: Integração de novas funcionalidades.

feature/*: Funcionalidades em desenvolvimento.

hotfix/*: Correções críticas em produção.

4. Reportar Bugs

Ao abrir uma Issue para reportar um bug, inclua:

Passos para reproduzir o problema.

Comportamento esperado vs. Comportamento atual.

Capturas de ecrã (se aplicável).
